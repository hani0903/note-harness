---
name: issue-reviewer
description: >
  이슈 문서(issues.md) 또는 GitHub issue를 리뷰할 때 사용하는 검토 스킬.

  사용자가 다음과 같은 요청을 하면 반드시 사용한다.

  * "이슈 리뷰해줘"
  * "이슈 분해가 잘 되었는지 확인해줘"
  * "수직 슬라이싱이 맞는지 봐줘"
  * "Acceptance Criteria 검토해줘"
  * "이슈 품질 확인해줘"
  * "이슈 작성했는데 피드백해줘"

  사용자가 명시적으로 스킬 이름을 언급하지 않아도
  이슈 검토 맥락이 감지되면 적극적으로 사용한다.

  이 스킬은 새로운 이슈를 생성하거나 수정하지 않는다.

  역할:

  * 요구사항 누락 여부 확인
  * 수직 슬라이싱 여부 검증
  * Acceptance Criteria의 테스트 가능성 검증
  * 의존성 방향 검증
  * 범위 과대 여부 검증
  * 모호한 표현 탐지

  출력은 반드시 다음 3개 섹션으로 구성한다.

  * Critical: 반드시 수정이 필요한 문제
  * Warning: 수정 권장 사항
  * Passed: 현재 기준을 충족하는 항목

  리뷰 결과는 체크리스트 기반으로 판단하며,
  근거 없는 취향성 피드백은 제공하지 않는다.

# Issue Review Checklist

## Vertical Slice

- [ ] 이 이슈만 완료해도 사용자에게 제공 가능한 기능이 존재하는가?
- [ ] API/UI/상태가 한 이슈 안에 포함되어 있는가?
- [ ] 다른 이슈 없이도 테스트 가능한가?

## Acceptance Criteria

- [ ] Given에 실제 데이터값이 있는가?
- [ ] When이 사용자 행동인가?
- [ ] Then이 검증 가능한 값인가?
- [ ] 모호한 단어가 없는가?

금지어:

- 정상적으로
- 적절히
- 올바르게
- 보여진다
- 처리된다

## Scope

- [ ] 하루 이내 구현 가능한가?
- [ ] 수정 범위가 과도하게 넓지 않은가?
- [ ] Hook/UI/API 변경이 이슈 크기에 비해 많지 않은가?

## Dependency

- [ ] 역방향 의존성이 없는가?
- [ ] 앞 이슈 결과가 다음 이슈 입력이 되는가?
