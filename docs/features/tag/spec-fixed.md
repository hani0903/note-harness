# 태그 기능 정의서 (확정)

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 기능 요구사항

- 노트에 태그를 추가할 수 있다.
- 노트에 추가된 태그를 삭제할 수 있다.
- 태그 목록을 노트 상세 화면에서 확인할 수 있다.

---

## 데이터 구조

### Note 타입

`tags` 필드를 `string[]` 필수 필드로 추가한다.

```ts
interface Note {
  id: string;
  title: string;
  content: string;
  tags: string[]; // 추가
  createdAt: string;
  updatedAt: string;
}
```

### 저장 방식

- 노트 객체 안에 문자열 배열로 직접 저장한다.
- 별도 태그 컬렉션 없음.

### db.json 마이그레이션

기존 노트에 `"tags": []` 필드를 수동으로 추가해야 한다.

---

## 입력 UX

### 태그 추가

- 입력창에 태그명을 타이핑한 후 **Enter** 키를 누르면 태그가 추가된다.
- 추가된 태그는 입력창 왼쪽에 칩(chip) 형태로 표시된다.
- 입력창 플레이스홀더: `"태그 입력 후 Enter"`

### 태그 삭제

- 칩에 **hover** 시 x 버튼이 나타난다.
- x 버튼 클릭 시 해당 태그가 제거된다.

### 배치

NoteEditor 내 본문 입력창 아래, 저장/취소 버튼 위에 위치한다.

```
[ 제목 입력 ]
[ 본문 입력 ]
─────────────────────────────
[ 태그1 ] [ 태그2 ] [ 입력창 ]
[ 저장 ] [ 취소 ]
```

---

## 유효성 검사

| 규칙             | 내용                                   |
| ---------------- | -------------------------------------- |
| 빈 문자열        | 추가하지 않음 (trim 후 빈 값이면 무시) |
| 최대 글자 수     | 태그 1개당 15자 초과 시 무시           |
| 최대 개수        | 노트당 최대 10개 초과 시 무시          |
| 중복             | 소문자 정규화 후 동일한 값이면 무시    |
| 유효성 실패 처리 | 조용히 early return (alert 없음)       |

---

## 정규화 규칙

- 저장 전 `trim() + toLowerCase()` 적용한다.
- 예: `" React "` → `"react"`, `"TypeScript"` → `"typescript"`

---

## 저장 시점

- 태그 변경(추가/삭제)은 **로컬 상태**에만 반영된다.
- **저장 버튼 클릭 시** 제목, 본문과 함께 `updateNote`로 일괄 전송한다.
- 저장 없이 나가면 변경 사항이 사라진다 (제목/본문과 동일한 동작).

---

## 태그 순서

- 입력한 순서를 유지한다 (정렬 없음).
